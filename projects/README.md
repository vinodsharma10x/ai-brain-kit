# Projects

One folder per active effort: a build, a launch, a client engagement. A project has an outcome and an end; ongoing stuff lives in `areas/`.

Create one with **"new project: (name)"** and Claude stamps it from `_templates/project.md`.

```
projects/
└── my-launch/
    └── README.md    status, next action, plan, log
```

When a project finishes, set `status: done` in its README. If you never want to see it again, move it to `archive/projects/`.
